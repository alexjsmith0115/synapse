# Synapse — Design Document

**Date:** 2026-03-07
**Status:** Approved

## Overview

Synapse is a LSP-powered, graph database-backed MCP tool that creates an agent-first, node-based representation of a codebase. It solves the core limitation of static AST tools (like CodeGraphContext) by using a Language Server Protocol client to accurately resolve implicit and abstract dependencies — dependency injection, interface implementations, abstract method overrides — that static parsing cannot see.

Summaries and learned context are stored as first-class graph data, written by the agent (via MCP tool) rather than generated by the tool itself. No external LLM API dependencies.

## Key Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Graph database | FalkorDB | Lighter than Neo4j, no external server required, Cypher-compatible |
| LSP integration | Copy `solidlsp` from Serena (MIT) | Battle-tested C# support, avoids Serena's full dependency footprint |
| Language support | C# (POC), TypeScript (next) | Validate architecture on a real C# codebase |
| Indexing strategy | Eager (LSP at index time, shut down after) | Queries are pure graph reads; no LSP latency at query time |
| Summary storage | Properties on node + `:Summarized` label | Single source of truth; queryable by type and by "has summary" |
| LLM integration | None | Agent drives all summarization; tool is pure infrastructure |
| File watching | `watchdog` + keep LSP running in watch mode | C# LSP startup is slow; keeping it alive makes watch mode responsive |

## Graph Schema

### Node Types

| Label | Key Properties |
|---|---|
| `Repository` | `path`, `language`, `last_indexed` |
| `Directory` | `path`, `name` |
| `File` | `path`, `name`, `language`, `last_indexed` |
| `Namespace` | `name`, `full_name` |
| `Class` | `name`, `full_name`, `kind` (class/interface/abstract/enum/record) |
| `Method` | `name`, `full_name`, `signature`, `is_abstract`, `is_static` |
| `Property` | `name`, `full_name`, `type_name` |
| `Field` | `name`, `full_name`, `type_name` |

All nodes that have been summarized by an agent receive a secondary `:Summarized` label and two additional properties: `summary` (text) and `summary_updated_at` (timestamp).

### Relationship Types

| Edge | From → To | Meaning |
|---|---|---|
| `CONTAINS` | Directory→Directory, Directory→File, File→Namespace, Namespace→Class, Class→Method/Property/Field | Structural containment |
| `CALLS` | Method→Method | Direct method invocation |
| `INHERITS` | Class→Class | Extends a base class |
| `IMPLEMENTS` | Class→Class | Implements an interface |
| `OVERRIDES` | Method→Method | Overrides a virtual/abstract method |
| `REFERENCES` | Method/Field→Class | Type usage (parameter, return type, field type) |

## Architecture

```
src/
  synapse/
    lsp/          # solidlsp copy, stripped to C# + extensible base
    indexer/      # LSP-powered graph builder (full + incremental)
    graph/        # FalkorDB abstraction (schema, upsert, queries)
    watcher/      # watchdog-based file monitor → incremental re-index
    mcp/          # MCP server + tool definitions
    cli/          # Typer CLI wrapping core service layer
```

The `indexer` and `graph` layers are the core. The MCP server and CLI are thin consumers of the same service layer — no business logic lives in either interface.

## Data Flow

### Full Index (`synapse index <path>`)

1. Start C# language server, open solution/workspace
2. Walk all `.cs` files via LSP workspace symbols
3. For each file: upsert `Directory` + `File` nodes, resolve all symbols, upsert symbol nodes with `CONTAINS` edges
4. For each class/method: resolve `INHERITS`, `IMPLEMENTS` via LSP go-to-definition on base types; resolve `CALLS` via LSP find-references on method bodies; resolve `OVERRIDES` via LSP hierarchy queries; upsert relationship edges
5. Shut down language server
6. Write `Repository` node with `last_indexed` timestamp

### Incremental Re-index (file change via watcher)

1. Language server stays running in watch mode
2. On file modified: delete all nodes and edges originating from that file; re-run full index steps 3–4 for that file only
3. On file deleted: delete all nodes and edges originating from that file

## MCP Tools

### Indexing & Management

| Tool | Args | Description |
|---|---|---|
| `index_project` | `path`, `language` | Full index of a project/solution |
| `list_projects` | — | All indexed projects with stats |
| `delete_project` | `path` | Remove project and all its nodes |
| `get_index_status` | `path` | Last indexed timestamp, node/edge counts |

### Graph Exploration

| Tool | Args | Description |
|---|---|---|
| `get_symbol` | `full_name` | Symbol node, properties, and direct relationships |
| `find_implementations` | `interface_name` | All concrete types implementing an interface |
| `find_callers` | `method_full_name` | All methods that call a given method |
| `find_callees` | `method_full_name` | All methods called by a given method |
| `get_hierarchy` | `class_name` | Full inheritance chain (up and down) |
| `search_symbols` | `query`, `kind?` | Text search across symbol names |

### Summaries

| Tool | Args | Description |
|---|---|---|
| `set_summary` | `full_name`, `content` | Write agent-authored summary to a node |
| `get_summary` | `full_name` | Read summary from a node |
| `list_summarized` | `project_path?` | All nodes with summaries |

### Advanced

| Tool | Args | Description |
|---|---|---|
| `execute_query` | `cypher` | Raw read-only Cypher query |
| `watch_project` | `path` | Start file watching for incremental updates |
| `unwatch_project` | `path` | Stop watching |

## CLI

```
synapse index <path> [--language csharp]
synapse watch <path>
synapse unwatch <path>
synapse delete <path>
synapse status [<path>]

synapse symbol <full_name>
synapse callers <method_full_name>
synapse callees <method_full_name>
synapse implementations <interface_name>
synapse hierarchy <class_name>
synapse search <query> [--kind class|method|property|field]

synapse summary get <full_name>
synapse summary set <full_name> <content>
synapse summary list [--project <path>]

synapse query <cypher>
```

## Testing Strategy

- **Global timeout:** `pytest-timeout` with a 10s default on all tests; integration tests opt in to longer timeouts explicitly (`@pytest.mark.timeout(60)`)
- **Unit tests** (`tests/unit/`): graph layer, schema, indexer logic — all LSP interactions mocked, no real language server process, no risk of hangs
- **Integration tests** (`tests/integration/`): full index of a C# project, asserting known symbols and relationships exist in the graph; LSP process wrapped in a context manager with explicit kill timeout
- **MCP tests** (`tests/mcp/`): MCP tool responses tested against a live indexed graph
- **LSP fixture tests** (`tests/fixtures/csharp/`): small test repo with known DI patterns, interfaces, and call graphs; assertions verify implicit relationships are correctly resolved
- File watcher tests use a short debounce and explicit `observer.stop()` in teardown to prevent background thread leaks
